## 1. Training Script – In‑Depth Walkthrough

The file `training/train.py` builds a 6‑class apparel classifier using **MobileNetV2** as a base. The approach is called **transfer learning**: take a network already trained on a huge dataset (ImageNet), remove its final classification layer, and attach a new head for our specific task.

### 1.1 Imports and Parameters
```python
import tensorflow as tf
from tensorflow.keras.preprocessing.image import ImageDataGenerator
from tensorflow.keras.applications import MobileNetV2
from tensorflow.keras.models import Model
from tensorflow.keras.layers import Dense, GlobalAveragePooling2D, Dropout
from tensorflow.keras.callbacks import EarlyStopping, ModelCheckpoint, ReduceLROnPlateau
import os
```
Key libraries:
- `ImageDataGenerator` – loads images from folders and applies real‑time augmentation.
- `MobileNetV2` – lightweight, fast, and pre‑trained model perfect for this task.
- Callbacks help training run smoothly (stop early if no improvement, save the best model, reduce learning rate).

Parameters:
```python
img_size = 224          # MobileNetV2 expects 224×224
batch_size = 32
num_classes = 6
epochs = 30
```
These are standard; you can change them if needed.

### 1.2 Data Generators
```python
train_datagen = ImageDataGenerator(
    rescale=1./255,
    rotation_range=20,
    width_shift_range=0.2,
    height_shift_range=0.2,
    shear_range=0.2,
    zoom_range=0.2,
    horizontal_flip=True
)
val_datagen = ImageDataGenerator(rescale=1./255)
```
- `rescale=1./255` converts pixel values from [0,255] to [0,1].
- Augmentation (rotation, shifts, zoom, flip) artificially increases the variety of the training set – helps prevent overfitting.
- Validation set is only rescaled, no augmentation.

`flow_from_directory` reads images from sub‑folders whose names become the class labels. **Important:** The classes are ordered alphabetically by folder name. For example:
```
dataset/train/
    dress/
    hoodie/
    jacket/
    pant/
    shirt/
    shoes/
```
The order will be `dress, hoodie, jacket, pant, shirt, shoes`. Our intended order was `Shirt, Pant, Hoodie, Jacket, Dress/Bodycon, Shoes` – this **mismatch** must be fixed later. (We’ll address it.)

### 1.3 Building the Model
```python
base_model = MobileNetV2(weights='imagenet', include_top=False, input_shape=(224,224,3))
base_model.trainable = False  # freeze base layers first
```
- `include_top=False` removes the original 1000‑class layer.
- Freezing prevents the pre‑trained weights from being destroyed during the first phase.

**Custom head:**
```python
x = base_model.output
x = GlobalAveragePooling2D()(x)
x = Dense(128, activation='relu')(x)
x = Dropout(0.5)(x)
predictions = Dense(6, activation='softmax')(x)
model = Model(inputs=base_model.input, outputs=predictions)
```
- `GlobalAveragePooling2D` converts the feature maps into a single vector.
- A dense layer with 128 neurons learns new patterns.
- Dropout randomly disables 50% of neurons during training to fight overfitting.
- Final softmax layer outputs a probability for each of the 6 classes.

### 1.4 Two‑Phase Training
**Phase 1** (only the new head is trainable):
```python
model.compile(optimizer='adam', loss='categorical_crossentropy', metrics=['accuracy'])
history = model.fit(train_generator, validation_data=val_generator, epochs=10, callbacks=...)
```
This warms up the randomly initialized head while the base stays frozen.

**Fine‑tuning (Phase 2):**
```python
base_model.trainable = True
for layer in base_model.layers[:100]:
    layer.trainable = False
```
Here we unfreeze the base but keep early layers frozen (they detect generic edges/textures). Only deeper layers (from index 100 onward) will be fine‑tuned.  
Then we re‑compile with a **very low learning rate** (`1e-5`) to avoid wrecking the pre‑trained weights:
```python
model.compile(optimizer=tf.keras.optimizers.Adam(learning_rate=1e-5), ...)
history_finetune = model.fit(...)
```
Finally the model is saved to `../backend/model/apparel_model.h5`.

## 2. 🐛 Critical Bug: Preprocessing Mismatch

The training code applies `rescale=1./255` (pixels → [0,1]). However, **MobileNetV2 was originally trained with a specific preprocessing function** that normalises pixels to the range [-1, 1] using mean subtraction. If you feed [0,1] images into a frozen base, the pre‑trained weights receive values far from what they expect – the accuracy will be **significantly lower** than it could be.

### The Fix
Use TensorFlow’s built‑in `preprocess_input` for MobileNetV2 **both in training and inference**. That function scales images exactly as the original training did.

**Update the data generators:**
```python
from tensorflow.keras.applications.mobilenet_v2 import preprocess_input

train_datagen = ImageDataGenerator(
    preprocessing_function=preprocess_input,   # replaces rescale
    rotation_range=20,
    ...  # all other augmentations stay the same
)
val_datagen = ImageDataGenerator(preprocessing_function=preprocess_input)
```
Remove `rescale=1./255` entirely – the `preprocessing_function` will be applied after augmentation, ensuring the image ranges match what MobileNetV2 expects.

**Update `backend/utils.py`** to use the same function:
```python
import numpy as np
from PIL import Image
from tensorflow.keras.applications.mobilenet_v2 import preprocess_input

def preprocess_image(image_file, target_size=(224,224)):
    img = Image.open(image_file).convert('RGB')
    img = img.resize(target_size)
    img_array = np.array(img, dtype=np.float32)   # make sure float
    img_array = np.expand_dims(img_array, axis=0)
    img_array = preprocess_input(img_array)       # apply correct scaling
    return img_array
```
Without this fix, the model will still work, but its predictions will be less reliable – this is a **missing piece** that can cost you 5‑15% accuracy.

## 3. Missing: Saving and Loading Class Label Order

The training script does **not** record the class order learned from the folders. The backend hard‑codes:
```python
class_names = ['Shirt', 'Pant', 'Hoodie', 'Jacket', 'Dress/Bodycon', 'Shoes']
```
But `flow_from_directory` returns classes in **alphabetical order**, which will be `dress, hoodie, jacket, pant, shirt, shoes` – a totally different mapping! The app would mislabel every image.

### Fix
After calling `flow_from_directory`, save the mapping:
```python
class_indices = train_generator.class_indices   # dict: {'dress':0, 'hoodie':1, ...}
# Convert to a list in correct order
idx_to_class = {v: k for k, v in class_indices.items()}
class_names = [idx_to_class[i] for i in range(len(idx_to_class))]
print("Class order:", class_names)
```
Then **export this list** so the backend can use it:
```python
import json
with open('class_names.json', 'w') as f:
    json.dump(class_names, f)
```
Copy `class_names.json` to the `backend/` folder.

In `backend/app.py`, load it:
```python
import json
with open('class_names.json', 'r') as f:
    class_names = json.load(f)
```
Now the labels will always match the training folders, regardless of naming.

## 4. Backend Serving – app.py & utils.py Explained

### 4.1 Loading the Model
```python
model_path = os.path.join('model', 'apparel_model.h5')
model = tf.keras.models.load_model(model_path)
```
The model is loaded **once** when the Flask app starts. This is efficient – it stays in RAM for all requests.

### 4.2 The Prediction Endpoint
```python
@app.route('/predict', methods=['POST'])
def predict():
    if 'file' not in request.files: ...
    file = request.files['file']
    ...
    img_array = preprocess_image(file)
    predictions = model.predict(img_array)[0]
    predicted_idx = predictions.argmax()
    category = class_names[predicted_idx]
    confidence = float(predictions[predicted_idx])
    return jsonify({'category': category, 'confidence': confidence})
```
- `model.predict` returns a batch of probabilities; `[0]` extracts the first (only) image’s output.
- `argmax()` gets the index of the highest probability.
- The corresponding class name and confidence are sent back as JSON.

### 4.3 Missing Error Handling
If the model file is missing, the app crashes immediately with a cryptic error. Add a friendly check:
```python
if not os.path.exists(model_path):
    raise FileNotFoundError(f"Model not found at {model_path}. Run training first or download a pre‑trained model.")
```
Also, invalid image files (e.g., corrupted JPEG) can cause `preprocess_image` to fail. The existing `try/except` in the route catches that and returns a 500 error – that’s fine, but you might want to return a more user‑friendly message.

## 5. Summary of All Fixes & Missing Code

| Issue | How to fix |
|-------|------------|
| **Preprocessing mismatch** | Use `preprocess_input` in training (via `preprocessing_function`) and in `utils.py`. |
| **Class order is not saved** | Extract `class_indices` from the generator and save to JSON; load it in `backend/app.py`. |
| **Model file missing handling** | Add an existence check with a clear error message. |
| **Hard‑coded number of base layers to unfreeze** (100) may not be optimal | Instead dynamically freeze a certain percentage: `fine_tune_at = int(len(base_model.layers) * 0.8)` and freeze layers before that index. But this is optional. |
| **Validation split if no `val` folder** | If you only have a training folder, use `validation_split=0.2` in `ImageDataGenerator` and subset parameters in `flow_from_directory`. |
| **No GPU acceleration configured** | TensorFlow will auto‑detect GPU if available. Ensure CUDA/cuDNN are installed for NVIDIA GPUs. |
