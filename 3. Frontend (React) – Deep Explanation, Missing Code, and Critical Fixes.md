## 1. App.js – Step‑by‑Step Explanation

### 1.1 Imports and Component Initialization
```jsx
import React, { useState } from 'react';
import axios from 'axios';
```
- `useState` allows us to create state variables inside the functional component.
- `axios` is a popular HTTP client that simplifies making requests and handling responses.

### 1.2 State Variables
```jsx
const [selectedFile, setSelectedFile] = useState(null);
const [preview, setPreview] = useState(null);
const [result, setResult] = useState(null);
const [loading, setLoading] = useState(false);
```
- `selectedFile` – the actual file object selected by the user.
- `preview` – a temporary object URL to display a preview of the image.
- `result` – object containing `category` and `confidence` (or `error`).
- `loading` – boolean used to disable the button and show "Analyzing..." text.

### 1.3 Handling File Selection (`onFileChange`)
```jsx
const onFileChange = (event) => {
    const file = event.target.files[0];
    setSelectedFile(file);
    setPreview(URL.createObjectURL(file));
    setResult(null);
};
```
- Grabs the first file from the input.
- Creates an in‑browser URL for the image (blob URL) so it can be displayed in an `<img>` tag without uploading.
- Clears any previous result because a new file was selected.

**Problem 1 – Missing file type validation:**  
The code does not check if the selected file is actually an image. A user could select a PDF and `URL.createObjectURL` would still create a URL, but the preview would be broken and the backend would return an error.

**Problem 2 – Memory leak:**  
Every time a file is chosen, a new object URL is created. Old URLs are never released, leading to memory leaks. Even when the component unmounts, these URLs persist until the page is closed.

### 1.4 Sending the Prediction Request (`predict`)
```jsx
const predict = async () => {
    if (!selectedFile) return;
    setLoading(true);
    const formData = new FormData();
    formData.append('file', selectedFile);

    try {
        const response = await axios.post('http://localhost:5000/predict', formData);
        setResult(response.data);
    } catch (error) {
        console.error(error);
        setResult({ error: 'Prediction failed. Is the backend running?' });
    }
    setLoading(false);
};
```
- Creates a `FormData` object to mimic a file upload form.
- Sends a POST request to the backend endpoint.
- On success, the returned JSON (`{category, confidence}`) is stored in `result`.
- On failure, a generic error message is set.

**Problem 3 – Race condition:**  
Imagine the user selects `shirt.jpg` and clicks **Predict**. While the request is in flight, they select `pant.jpg`. The `onFileChange` handler clears `result`, but `loading` is still `true` from the first request. The second request is never triggered automatically (user must click again). However, if the user rapidly changes files and clicks predict multiple times, older requests could complete after newer ones, displaying the wrong result. More critically, the UI becomes “stuck” in loading state if they change file without clicking again.

**Problem 4 – Hardcoded backend URL:**  
`'http://localhost:5000/predict'` will only work locally. If you deploy the frontend to Vercel or Netlify, you’ll need to change this manually. A better approach is to use an environment variable.

**Problem 5 – Only text loading indicator:**  
The button text changes to “Analyzing…”, but there is no visual spinner or disabled styling (the button is disabled, but the user might miss it). A professional UI usually shows a spinner animation.

## 2. Fixes – The Improved `App.js`

Below is a rewritten version of `App.js` that addresses all five issues. Comments explain each improvement.

```jsx
import React, { useState, useEffect, useRef } from 'react';
import axios from 'axios';
import './App.css';  // We'll move styles to a CSS file for cleanliness

function App() {
  const [selectedFile, setSelectedFile] = useState(null);
  const [preview, setPreview] = useState(null);
  const [result, setResult] = useState(null);
  const [loading, setLoading] = useState(false);

  // 1. AbortController to cancel previous request on new file selection
  const abortControllerRef = useRef(null);

  // 2. Revoke object URL when component unmounts or when a new preview is created
  useEffect(() => {
    // clean up on unmount
    return () => {
      if (preview) URL.revokeObjectURL(preview);
    };
  }, []); // run only once on mount/unmount

  const onFileChange = (event) => {
    const file = event.target.files[0];
    if (!file) return;

    // 3. Validate file type (allow only images)
    if (!file.type.startsWith('image/')) {
      alert('Please select a valid image file.');
      return;
    }

    // Revoke old preview URL to free memory
    if (preview) URL.revokeObjectURL(preview);

    // 4. Cancel any ongoing prediction request
    if (abortControllerRef.current) {
      abortControllerRef.current.abort();
    }

    setSelectedFile(file);
    setPreview(URL.createObjectURL(file));
    setResult(null);
    setLoading(false);   // reset loading state if a new file is selected
  };

  const predict = async () => {
    if (!selectedFile) return;

    // Create a new AbortController for this request
    const controller = new AbortController();
    abortControllerRef.current = controller;

    setLoading(true);
    const formData = new FormData();
    formData.append('file', selectedFile);

    try {
      // 5. Use environment variable for API URL
      const apiUrl = process.env.REACT_APP_API_URL || 'http://localhost:5000/predict';
      const response = await axios.post(apiUrl, formData, {
        signal: controller.signal,
      });
      setResult(response.data);
    } catch (error) {
      if (axios.isCancel(error)) {
        console.log('Request canceled:', error.message);
        return; // do nothing, the request was canceled intentionally
      }
      // 6. Distinguish between network error and other errors
      if (error.response) {
        // Server responded with a status outside 2xx
        setResult({ error: `Server error: ${error.response.data.error || 'Unknown'}` });
      } else if (error.request) {
        setResult({ error: 'Cannot reach the server. Is it running?' });
      } else {
        setResult({ error: 'An unexpected error occurred.' });
      }
    } finally {
      setLoading(false);
    }
  };

  // 7. Custom loading spinner component (inline, could be extracted)
  const Spinner = () => (
    <div className="spinner"></div>
  );

  return (
    <div className="container">
      <h1 className="title">👕 Apparel Classifier</h1>
      <div className="card">
        <input
          type="file"
          accept="image/*"   // HTML5 validation (complements our JS check)
          onChange={onFileChange}
          className="fileInput"
        />
        {preview && (
          <div className="previewContainer">
            <img src={preview} alt="preview" className="previewImage" />
          </div>
        )}
        <button
          onClick={predict}
          disabled={!selectedFile || loading}
          className="button"
        >
          {loading ? <Spinner /> : 'Predict Clothing'}
        </button>

        {result && (
          <div className="resultBox">
            {result.error ? (
              <p className="error">{result.error}</p>
            ) : (
              <>
                <h2 className="category">{result.category}</h2>
                <p className="confidence">
                  Confidence: {(result.confidence * 100).toFixed(2)}%
                </p>
                <progress value={result.confidence} className="progress" />
              </>
            )}
          </div>
        )}
      </div>
    </div>
  );
}

export default App;
```

## 4. Setting the Environment Variable

- Create a file `frontend/.env` in the root of your React project:
  ```
  REACT_APP_API_URL=http://localhost:5000/predict
  ```
- In production, set it to your deployed backend URL.
- Note: React requires variables to be prefixed with `REACT_APP_`.

## 5. Testing the Frontend Without the Backend

You can test the UI and logic without a running backend by mocking `axios`. Install `axios-mock-adapter` for development:

```bash
npm install axios-mock-adapter --save-dev
```

Then temporarily modify `App.js` (or a test setup) to intercept calls:

```jsx
import MockAdapter from 'axios-mock-adapter';

// Inside component, before render:
useEffect(() => {
  const mock = new MockAdapter(axios);
  mock.onPost('http://localhost:5000/predict').reply(200, {
    category: 'Hoodie',
    confidence: 0.9432,
  });
  return () => mock.restore();
}, []);
```

This way you can verify file upload, preview, loading spinner, and success display.
