- Images must be **RGB** (3 channels), of any size (they will be resized to 224√ó224 during training).

- Classes must be balanced unless you apply class‚Äëweight strategies (we will cover that).

## 2. Option 1: Kaggle ‚Äì Fashion Product Images (Recommended)

The **Fashion Product Images** dataset on Kaggle is the most straightforward source.  
It contains ~44,000 images of apparel and accessories with a CSV file mapping each image to a category and sub‚Äëcategory.

### 2.1 How to Download
1. Go to [Kaggle Fashion Product Images](https://www.kaggle.com/datasets/paramaggarwal/fashion-product-images-dataset) (create a free Kaggle account if needed).
2. Click **Download** (you may need to accept competition rules). The zip file is about 2.5 GB.
3. Unzip it. You‚Äôll get a folder `fashion-dataset` containing:
   - `images/` ‚Äì all product photos.
   - `styles.csv` ‚Äì metadata: `id`, `gender`, `masterCategory`, `subCategory`, `articleType`, `baseColour`, etc.

### 2.2 Filtering for Our 6 Classes
We need images that belong to specific `articleType` or `subCategory` values.  
Example mapping (you may need to inspect the CSV to finalise):

| Our Class       | CSV `articleType` values to include                              |
|-----------------|------------------------------------------------------------------|
| Shirt           | `'Shirts'`                                                       |
| Pant            | `'Trousers'`, `'Jeans'`                                          |
| Hoodie          | `'Hoodie'`                                                       |
| Jacket          | `'Jacket'`, `'Blazer'`                                           |
| Dress/Bodycon   | `'Dress'`, `'Bodycon Dress'`, `'Maxi Dress'`, `'Summer Dress'`   |
| Shoes           | `'Shoes'`, `'Sandals'`, `'Sneakers'`, `'Boots'`, `'Heels'`       |

**Python script to extract and organise:**
```python
import pandas as pd
import os
import shutil
from sklearn.model_selection import train_test_split

# Paths
csv_path = 'fashion-dataset/styles.csv'
images_dir = 'fashion-dataset/images'
output_base = 'training/dataset'

# Define class mapping
class_map = {
    'Shirt': ['Shirts'],
    'Pant': ['Trousers', 'Jeans'],
    'Hoodie': ['Hoodie'],
    'Jacket': ['Jacket', 'Blazer'],
    'Dress/Bodycon': ['Dress', 'Bodycon Dress', 'Maxi Dress', 'Summer Dress'],
    'Shoes': ['Shoes', 'Sandals', 'Sneakers', 'Boots', 'Heels']
}

# Load CSV
df = pd.read_csv(csv_path, usecols=['id', 'articleType'])

# Filter rows and create folder structure
for class_name, article_types in class_map.items():
    # Filter image IDs
    mask = df['articleType'].isin(article_types)
    ids = df.loc[mask, 'id'].astype(str).tolist()
    
    # Create train/val folders
    os.makedirs(f'{output_base}/train/{class_name}', exist_ok=True)
    os.makedirs(f'{output_base}/val/{class_name}', exist_ok=True)
    
    # Split ids (80% train, 20% val)
    train_ids, val_ids = train_test_split(ids, test_size=0.2, random_state=42)
    
    # Copy images
    for img_id in train_ids:
        src = os.path.join(images_dir, f'{img_id}.jpg')
        if os.path.exists(src):
            shutil.copy(src, f'{output_base}/train/{class_name}/')
    for img_id in val_ids:
        src = os.path.join(images_dir, f'{img_id}.jpg')
        if os.path.exists(src):
            shutil.copy(src, f'{output_base}/val/{class_name}/')

print("Dataset prepared successfully.")
```
This script automatically creates the required `train/` and `val/` subfolders with the correct class names. You can now run `train.py` from `training/`.

## 3. Option 2: DeepFashion (Large‚ÄëScale Clothes Database)

[DeepFashion](http://mmlab.ie.cuhk.edu.hk/projects/DeepFashion.html) (the Category and Attribute Prediction Benchmark) offers over 200,000 images with rich annotations. It is ideal if you want a larger variety.

### How to Use
- Download the **Category and Attribute Prediction** subset (images + `list_category_img.txt`, `list_category_cloth.txt`).
- The file `list_category_cloth.txt` maps category IDs to names; `list_category_img.txt` maps each image to a category ID.
- Our 6 classes can be mapped by picking category names like `'Blouse'`, `'Pants'`, `'Sweater'` (for hoodie/jacket), etc. You‚Äôll need to manually inspect the 50 categories and decide which to include.
- Once mapped, write a similar script to copy images into `training/dataset/train` and `val`.

*Caution:* The download is >3 GB and requires permission for non‚Äëcommercial use.

## 4. Option 3: Web Scraping (For Supplemental Data)

If you need extra images for a specific class, you can programmatically download from free‚Äëto‚Äëuse APIs.

### 4.1 Unsplash API
- Unsplash allows free use of images (check license). You can search for ‚Äúshirt‚Äù, ‚Äúsneakers‚Äù, etc., and download up to 50 images per hour on the free tier.
- Use a Python script with `requests` and the Unsplash API.

### 4.2 Pexels API
- Similar, returns stock photos of clothing. Often more lifestyle shots.

### 4.3 Bing Image Search
- Microsoft Azure offers a Bing Image Search API with a free tier (1000 transactions/month).
- Query ‚Äúshirt clothing‚Äù and download up to 150 images per query. Filter by license (public domain / free to share and use).

**Important:** Always respect robots.txt and API rate limits. For a learning project, downloading a few hundred images is generally acceptable.

## 5. What If I Only Have a Tiny Dataset? (Data Augmentation & Synthetic Data)

If you cannot source hundreds of images per class, you can **still train a viable model** by aggressively augmenting a small initial set.

- **Start with as few as 30‚Äì50 images per class** (e.g., from your own wardrobe photos).
- Use the `ImageDataGenerator` we already configured (rotation, shift, shear, zoom, flip). This multiplies your data manifold significantly.
- For even more variety, consider **Gaussian noise injection**:
  ```python
  def add_noise(img):
      noise = np.random.normal(0, 0.1, img.shape)
      return np.clip(img + noise, 0.0, 1.0)
  ```
  Add this as a custom `preprocessing_function` *in addition* to the main one, but be careful to chain it correctly.
- **CutMix / MixUp** ‚Äì advanced augmentation techniques that can be added via `tensorflow.keras.layers.RandomCrop` or custom generators. (Not necessary for initial version.)

**Note:** The model may overfit heavily on small data. Use stronger dropout (0.7) and early stopping with small `patience`. Also fine‚Äëtune fewer layers.

## 6. Data Splitting Strategy

We use a fixed random split (80% train, 20% val) as shown in the script above. For a more robust evaluation, especially with small data, use **k‚Äëfold cross validation**, but for simplicity we‚Äôll stick to a single split.

**Balancing:** If one class has 1000 images and another only 200, the model will bias toward the larger class. Fix it with:
```python
class_weights = compute_class_weight('balanced', classes=np.unique(labels), y=labels)
```
and pass `class_weight=class_weights` in `model.fit()`. The `train.py` does not include this ‚Äì **it‚Äôs a recommended addition**.
