### 1.2 Unit Test for Image Preprocessing (`test_utils.py`)
**File:** `backend/tests/test_utils.py`

We test that `preprocess_image` accepts a file‑like object and returns a NumPy array with the correct shape and value range.

```python
import pytest
import numpy as np
from PIL import Image
import io
from backend.utils import preprocess_image   # adjust import path as needed

def create_dummy_image_file(mode='RGB', size=(300, 300)):
    """Create a dummy file-like object with a coloured image."""
    img = Image.new(mode, size, color=(100, 150, 200))
    buf = io.BytesIO()
    img.save(buf, format='JPEG')
    buf.seek(0)
    return buf

def test_preprocess_image_shape():
    file = create_dummy_image_file()
    img_array = preprocess_image(file, target_size=(224, 224))
    assert img_array.shape == (1, 224, 224, 3)  # batch, height, width, channels

def test_preprocess_image_range():
    """After preprocess_input, values are roughly in [-1, 1]."""
    file = create_dummy_image_file()
    img_array = preprocess_image(file)
    # MobileNetV2's preprocess_input scales from [0,255] to [-1,1]
    assert img_array.min() >= -1.5 and img_array.max() <= 1.5

def test_preprocess_image_handle_corrupt():
    """Should raise an exception on non-image file."""
    bad_file = io.BytesIO(b"this is not an image")
    with pytest.raises(Exception):
        preprocess_image(bad_file)
```

**Run:** `pytest tests/test_utils.py -v`

### 1.3 Testing the `/predict` Endpoint (`test_app.py`)

We use Flask’s test client and mock `model.predict` so we don't need a real trained model.

**File:** `backend/tests/test_app.py`

```python
import pytest
import io
import json
from unittest.mock import patch, MagicMock
from backend.app import app   # import the Flask app (but we'll mock model loading)

# Mock the model loading before importing app
# We'll do a trick: use a fixture to mock tensorflow.keras.models.load_model
# But since app loads the model at import time, we need to mock before import.
# Solution: import app inside a test function with patching.
# Alternatively, restructure app.py to load model lazily, but we can patch.

@pytest.fixture
def client():
    # Mock the model
    mock_model = MagicMock()
    # fake prediction returning high prob for class 2 (index 2)
    mock_model.predict.return_value = [[0.01, 0.02, 0.9, 0.03, 0.02, 0.02]]
    with patch('tensorflow.keras.models.load_model', return_value=mock_model):
        # Re-import app after mock, so the global model is our mock
        from backend import app as flask_app
        flask_app.config['TESTING'] = True
        with flask_app.test_client() as client:
            yield client, mock_model

def create_test_image_file():
    """Create a real JPEG image in memory."""
    from PIL import Image
    img = Image.new('RGB', (100, 100), color='red')
    buf = io.BytesIO()
    img.save(buf, format='JPEG')
    buf.seek(0)
    return buf

def test_predict_no_file(client_fixture):
    client, _ = client_fixture
    response = client.post('/predict', data={})
    assert response.status_code == 400
    assert 'error' in response.get_json()

def test_predict_success(client_fixture):
    client, mock_model = client_fixture
    mock_model.predict.return_value = [[0.01, 0.02, 0.9, 0.03, 0.02, 0.02]]
    img_file = create_test_image_file()
    data = {'file': (img_file, 'test.jpg')}
    response = client.post('/predict', content_type='multipart/form-data', data=data)
    assert response.status_code == 200
    result = response.get_json()
    assert 'category' in result
    assert 'confidence' in result
    # confidence should be 0.9 (the mock value)
    assert abs(result['confidence'] - 0.9) < 1e-6

def test_predict_class_mapping(client_fixture):
    """Check that the correct class name is returned (based on mock)."""
    client, mock_model = client_fixture
    # Make class 2 the highest -> should map to the class_names[2]
    from backend.app import class_names
    mock_model.predict.return_value = [[0.1, 0.2, 0.5, 0.05, 0.05, 0.1]]
    img_file = create_test_image_file()
    data = {'file': (img_file, 'test.jpg')}
    response = client.post('/predict', content_type='multipart/form-data', data=data)
    result = response.get_json()
    assert result['category'] == class_names[2]   # depends on order
```

**Important:** To avoid the global model loading error, we mock `load_model` before importing `app`. The fixture above does that. If you prefer, refactor `app.py` to load the model inside a function (lazy loading), but for now this mocking approach works.

**Run:** `pytest tests/test_app.py -v`

### 1.4 Integration Test with a Real (or Dummy) Model

For a full integration test, we can train a tiny model on synthetic data and then test the endpoint. This verifies the entire backend pipeline including actual TensorFlow prediction.

**File:** `backend/tests/test_integration.py`

```python
import pytest
import io
import os
import tensorflow as tf
import numpy as np
from PIL import Image
import tempfile
from backend.app import app as flask_app  # This time we'll load a real model

@pytest.fixture(scope="module")
def trained_model_path():
    # Create a tiny dataset and train a minimal model, save it
    # For simplicity, just create a model with random weights and save it
    os.makedirs('backend/model', exist_ok=True)
    model = tf.keras.Sequential([
        tf.keras.layers.Input(shape=(224,224,3)),
        tf.keras.layers.Flatten(),
        tf.keras.layers.Dense(6, activation='softmax')
    ])
    model.compile(optimizer='adam', loss='categorical_crossentropy')
    model_path = 'backend/model/apparel_model.h5'
    model.save(model_path)
    # Also create a class_names.json for this test
    import json
    with open('class_names.json', 'w') as f:
        json.dump(['Shirt', 'Pant', 'Hoodie', 'Jacket', 'Dress/Bodycon', 'Shoes'], f)
    yield model_path
    # Cleanup
    os.remove(model_path)
    if os.path.exists('class_names.json'):
        os.remove('class_names.json')

def test_real_prediction_integration(trained_model_path):
    # Now start the app (it will load the real model)
    flask_app.config['TESTING'] = True
    with flask_app.test_client() as client:
        img = Image.new('RGB', (224,224), color='green')
        buf = io.BytesIO()
        img.save(buf, format='JPEG')
        buf.seek(0)
        data = {'file': (buf, 'test.jpg')}
        response = client.post('/predict', content_type='multipart/form-data', data=data)
        assert response.status_code == 200
        json_data = response.get_json()
        assert 'category' in json_data
        assert 'confidence' in json_data
```

This test runs a real TensorFlow model (random weights) and checks that no exception occurs. It’s slower but gives confidence.

## 2. Frontend Testing (React)


### 2.2 Test File (`frontend/src/App.test.js`)

Replace the default test file with:

```jsx
import React from 'react';
import { render, screen, fireEvent, waitFor } from '@testing-library/react';
import '@testing-library/jest-dom';
import axios from 'axios';
import MockAdapter from 'axios-mock-adapter';
import App from './App';

describe('App Component', () => {
  let mock;

  beforeAll(() => {
    mock = new MockAdapter(axios);
  });

  afterEach(() => {
    mock.reset();
  });

  afterAll(() => {
    mock.restore();
  });

  test('renders heading and file input', () => {
    render(<App />);
    expect(screen.getByText(/👕 Apparel Classifier/i)).toBeInTheDocument();
    expect(screen.getByLabelText(/choose file/i) || screen.getByRole('button', { name: /predict/i })).toBeTruthy();
  });

  test('allows file selection and shows preview', async () => {
    render(<App />);
    // Create a mock file
    const file = new File(['dummy content'], 'shirt.png', { type: 'image/png' });
    const input = document.querySelector('input[type="file"]');
    // Simulate file drop/selection
    Object.defineProperty(input, 'files', { value: [file] });
    fireEvent.change(input);
    // Preview image should appear
    await waitFor(() => {
      const img = document.querySelector('img');
      expect(img).toBeInTheDocument();
    });
  });

  test('calls predict and displays result', async () => {
    mock.onPost('http://localhost:5000/predict').reply(200, {
      category: 'Hoodie',
      confidence: 0.95,
    });

    render(<App />);
    // Simulate file selection first
    const file = new File(['dummy'], 'hoodie.jpg', { type: 'image/jpeg' });
    const input = document.querySelector('input[type="file"]');
    Object.defineProperty(input, 'files', { value: [file] });
    fireEvent.change(input);
    
    // Click predict button
    const button = screen.getByRole('button', { name: /predict clothing/i });
    fireEvent.click(button);

    // Wait for result
    await waitFor(() => {
      expect(screen.getByText('Hoodie')).toBeInTheDocument();
      expect(screen.getByText(/95.00%/)).toBeInTheDocument();
    });
  });

  test('shows error when backend fails', async () => {
    mock.onPost('http://localhost:5000/predict').networkError();

    render(<App />);
    const file = new File(['dummy'], 'shirt.jpg', { type: 'image/jpeg' });
    const input = document.querySelector('input[type="file"]');
    Object.defineProperty(input, 'files', { value: [file] });
    fireEvent.change(input);
    fireEvent.click(screen.getByRole('button', { name: /predict clothing/i }));

    await waitFor(() => {
      expect(screen.getByText(/Cannot reach the server/i)).toBeInTheDocument();
    });
  });

  test('disables button and shows spinner while loading', async () => {
    // Use a delayed response to check loading state
    mock.onPost('http://localhost:5000/predict').reply(() =>
      new Promise(resolve => setTimeout(() => resolve([200, { category: 'Shirt', confidence: 0.8 }]), 100))
    );

    render(<App />);
    const file = new File(['dummy'], 'shirt.jpg', { type: 'image/jpeg' });
    const input = document.querySelector('input[type="file"]');
    Object.defineProperty(input, 'files', { value: [file] });
    fireEvent.change(input);
    const button = screen.getByRole('button', { name: /predict clothing/i });
    fireEvent.click(button);

    // The button should be disabled and show spinner (the spinner class)
    expect(button).toBeDisabled();
    expect(button.querySelector('.spinner')).toBeInTheDocument();

    // Wait for result and check that spinner is gone
    await waitFor(() => {
      expect(screen.getByText('Shirt')).toBeInTheDocument();
    });
    expect(button.querySelector('.spinner')).not.toBeInTheDocument();
  });
});
```

**Run:** `npm test` (or `npx react-scripts test`). All tests should pass.

## 3. Testing the Training Script (Optional but Recommended)

```python
import os
import shutil
import tempfile

# For demonstration, we can test that generators work with a tiny dummy dataset

def test_data_generators():
    # Create temporary directory structure
    tmp_dir = tempfile.mkdtemp()
    for split in ['train', 'val']:
        for class_name in ['shirt', 'pant']:
            os.makedirs(f'{tmp_dir}/{split}/{class_name}', exist_ok=True)
            # Create a single dummy image
            img = tf.keras.preprocessing.image.random_shear(
                tf.keras.preprocessing.image.random_rotation(
                    tf.random.uniform((224,224,3), 0, 255)
                ), 0.2
            )
            # Save as PNG
            tf.keras.preprocessing.image.save_img(
                f'{tmp_dir}/{split}/{class_name}/test.png', img
            )
    # Now test the generator
    from tensorflow.keras.preprocessing.image import ImageDataGenerator
    train_gen = ImageDataGenerator(rescale=1./255).flow_from_directory(
        f'{tmp_dir}/train', target_size=(224,224), batch_size=2, class_mode='categorical'
    )
    batch = next(train_gen)
    assert batch[0].shape == (2,224,224,3)
    shutil.rmtree(tmp_dir)
```

## 4. Manual End‑to‑End Testing

After unit and integration tests pass, perform a manual test:

1. **Train a model** (or use a pre‑trained one) and place `apparel_model.h5` in `backend/model/`.
2. **Start the backend:** `cd backend && python app.py`. Verify `http://localhost:5000` is reachable (no route needed).
3. **Start the frontend:** `cd frontend && npm start`. Open `http://localhost:3000`.
4. **Upload a sample image** (e.g., a photo of a shirt from your computer).
5. **Click Predict** and check:
   - The image preview is shown.
   - The loading spinner appears.
   - A category and confidence percentage appear within a few seconds.
   - No errors in the browser console or backend terminal.
6. **Test error cases:**
   - Stop the backend and try predicting – an error message should appear.
   - Upload a non‑image file (like a `.txt`) – frontend should reject it (file type validation) or backend returns an error.

## 5. Continuous Integration (CI) Tips

You can automate all tests using GitHub Actions (or any CI service). A simple workflow:

- **Backend tests:** run on Python 3.8+, install requirements, run `pytest`.
- **Frontend tests:** run on Node 16+, `npm ci`, `npm test -- --watchAll=false`.
- **Integration tests** may require TensorFlow; they can run on GitHub‑hosted runners but may be slower.
